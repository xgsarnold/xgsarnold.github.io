My Experience

CAN<i>WALK

April 2016

In the final three weeks of the Iron Yard my team of two backend and two frontend engineers developed a webapp with an Angular UI and Rails API. The app allows users to enter a destination in order to develop a route based on their current destination with details regarding weather, ease, accessibility, enjoyability, distance and time. After completing a walk the user can rate the route. CAN<i>WALK offers street, map, or satellite view and populates the route with favorite places of users by comparing previous routes of others. These routes can be revisited from the user dashboard. If a user is unsure where they might like to walk, the app generates suggestions based on current location and distance preferences of users. To accomplish this from the backend, we consumed the Weather Underground API and rendered filtered SQL queries of our database as an API for the front end to consume. Other backend services include mailers for password resetting and email conformation beyond security precautions managed through token authentication.
